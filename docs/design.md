# Prism
## 初步设计文档 v0.1

---

# 1. 项目需求描述

## 1.1 项目背景

日常学习和工作过程中，会积累大量信息资源：

- 技术教程
- 博客文章
- 知乎/CSDN等平台内容
- 官方文档
- GitHub项目
- 个人笔记

目前这些信息分散在：

- 浏览器收藏夹
- 不同网站收藏系统
- 本地笔记软件

导致：

- 信息难以统一管理
- 收藏内容容易丢失
- 网页资源和个人笔记之间缺少关联
- 后续查找成本较高


## 1.2 项目目标

开发一个个人使用的信息管理工具，用于：

- 统一收集外部信息
- 管理个人知识资产
- 建立网页资源与笔记之间的关联
- 提供快速检索能力

软件定位：

> 作为个人知识系统的索引和管理层，不替代已有工具（如 Obsidian）。

---

# 2. 功能描述

## 2.1 网页资源管理

用于收集和管理互联网资源。

功能：

- 保存网页链接
- 自动获取网页标题
- 添加备注
- 添加标签（Tag）
- 分组管理
- 搜索网页
- 查看收藏时间
- 快速访问网页


目标：

替代浏览器收藏夹，提供更强的信息整理能力。


---

## 2.2 Obsidian笔记管理

用于管理已有 Obsidian 知识库。

功能：

- 读取本地 Obsidian Vault
- 扫描 Markdown 文件
- 建立笔记索引
- 查看笔记列表
- 按标签、时间、分类等方式管理视图
- 搜索笔记
- 点击后调用 Obsidian 打开对应文件


原则：

- 不修改 Obsidian 数据
- 不替代 Obsidian 编辑功能
- 只提供额外管理视图


---

## 2.3 统一搜索

提供统一的信息检索入口。

搜索范围：

- 网页标题
- 网页备注
- 标签
- Obsidian笔记标题
- 笔记内容


---

## 2.4 后续扩展方向

可能增加：

- GTD任务管理
- 网页离线保存
- AI摘要
- 自动标签生成
- 信息关联图谱
- 项目管理
- 多设备同步


---

# 3. 技术选型

## 3.1 应用形态

采用桌面客户端。

原因：

- 需要访问本地文件
- 需要管理 Obsidian Vault
- 需要本地数据存储
- 方便长期个人使用


---

## 3.2 技术栈

### 桌面框架

Electron

原因：

- 开发效率高
- 生态成熟
- 适合个人工具开发
- 支持跨平台


### 前端

React + TypeScript

原因：

- 开发效率高
- UI生态丰富
- 方便快速迭代


### 构建工具

Vite

原因：

- 开发启动快
- 配置简单


### UI

Tailwind CSS + shadcn/ui

原因：

- 快速构建现代化界面
- 避免传统桌面软件UI风格


### 数据存储

SQLite

原因：

- 本地数据库
- 单文件存储
- 易迁移
- 适合个人数据规模


### 笔记格式

Markdown

原因：

- 开放格式
- 与Obsidian兼容
- 数据长期可用


### 搜索

SQLite FTS

原因：

- 简单
- 性能足够
- 无需额外服务


### 浏览器扩展

Chrome Extension

用于：

- 快速保存网页
- 与桌面客户端通信


---

# 4. 设计原则

1. 数据属于用户，避免软件锁定

2. 优先支持开放格式

3. 快速收集，后续整理

4. 不重复造已有工具

5. 保持轻量，支持长期迭代
