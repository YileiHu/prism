# Feature: 收藏集 (Collections)

> 状态: **已实现**

## 概述

纯视图层的笔记分类组织，不修改原始 Markdown 文件。支持收藏集内二级分组、拖拽排序、批量操作、右键菜单。数据以 JSON 文件持久化在知识库目录内，随仓库自然迁移。

---

## 数据存储

### `.prism/` 目录

参考 Obsidian 的 `.obsidian/` 设计，在知识库根目录下创建 `.prism/` 存放该仓库维度的视图数据：

```
<vaultPath>/
├── .prism/
│   └── collections.json    ← 收藏集 + 分组数据 (version 2)
├── notes/
└── ...
```

| 存储位置 | 内容 | 迁移 |
|----------|------|------|
| SQLite `settings` | 全局配置（知识库列表、外部程序路径、主题） | 绑定机器 |
| `.prism/collections.json` | 收藏集、分组、排序 | 随仓库文件迁移 |

### `collections.json` (v2)

```json
{
  "version": 2,
  "collections": [
    {
      "id": "a1b2c3d4",
      "name": "算法",
      "notePaths": ["未分组的笔记.md"],
      "groups": [
        {
          "id": "g1",
          "name": "经典算法",
          "notePaths": ["笔记/分治法.md", "笔记/动态规划.md"]
        }
      ]
    }
  ]
}
```

- `notePaths` — 未分组笔记（v1 兼容）。拖入收藏集的笔记默认放这里
- `groups` — 二级分组（v2 新增）。每个分组有独立 `notePaths` 数组
- 路径统一使用 `/` 分隔符，相对路径，跨平台适配
- 数组顺序即展示/拖拽顺序

### 读写策略

- 切换知识库时并行加载 `scanVault` + `collections:load`
- 每次变更后 300ms 防抖自动写入
- JSON 损坏 → 返回 null + 备份为 `.bak`

## 界面布局

```
┌──────────┬──────────────────────────────┐
│ Sidebar  │ [🔍 搜索...]          [选择] │  ← 仅全部笔记视图
│          ├──────────────────────────────┤
│ 收藏集 1  │ 收藏集详情：                  │
│ 收藏集 2  │ ┌─ 经典算法 ── 4 ──── [⋯] ┐│  ← 分组卡片
│          │ │ ≡ 分治法                  ││
│          │ │ ≡ 动态规划                ││
│          │ └──────────────────────────┘│
│          │ ┌─ 未分组 ── 1 ────────────┐│  ← 可折叠
│          │ │ ≡ 图论                    ││
│          │ └──────────────────────────┘│
│          │ [+ 添加分组]                 │
├──────────┤                              │
│[📁][🔄] │                              │  ← 底部工具栏
│[+] [🏠] │                              │
└──────────┴──────────────────────────────┘
```

- **全部笔记**: 纯平铺列表，无文件夹树形结构
- **收藏集详情**: 分组卡片 + 未分组卡片，均支持折叠
- 侧边栏底部 4 个图标按钮：选仓库 / 刷新 / 新建收藏集 / 全部笔记

### 分组卡片配色

5 色复古循环（仅用于分组标题的竖条和组名）：
砖红 → 芥末黄 → 灰蓝 → 橄榄绿 → 循环

卡片底色、笔记文字均无颜色，保持简洁。

## 交互

| 操作 | 方式 |
|------|------|
| 创建收藏集 | 侧边栏底部 `+` 按钮 → 弹窗输入名称 |
| 重命名/删除收藏集 | 悬停 `⋯` → 重命名 / 删除（确认弹窗） |
| 拖拽笔记到收藏集 | 从全部笔记列表拖到侧边栏收藏集上 |
| 收藏集排序 | 侧边栏内拖拽收藏集整行 |
| 右键菜单 | Obsidian 打开 / 文件管理器定位 / 添加到收藏集 / 移动到回收站 |
| 删除文件 | 右键或批量 → 确认弹窗 → `shell.trashItem` |
| 批量操作 | 点击"选择" → 勾选笔记 → 底部操作栏（批量添加到收藏集 / 批量删除） |
| 分组内笔记排序 | `≡` 手柄拖拽 |
| 笔记跨组移动 | 拖到目标分组卡片上 |
| 展开/折叠全部 | 收藏集标题栏右侧按钮 |

---

## 自动分组

运行 `scripts/generate-collections.mjs` 可根据 Obsidian 的 `.base` 文件（Datacore 视图配置）自动生成收藏集和二级分组。分组依据：笔记标签去掉收藏集匹配标签后，用剩余第一个标签作为分组名。

## 新增 IPC

| Channel | 用途 |
|---------|------|
| `collections:load` | 读取 `<vault>/.prism/collections.json` |
| `collections:save` | 写入 `<vault>/.prism/collections.json` |
| `shell:show-item-in-folder` | 文件管理器中定位文件 |
| `shell:trash-file` | 移动单个文件到回收站 |
| `shell:trash-files` | 批量移动文件到回收站 |
